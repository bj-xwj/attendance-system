# 员工考勤管理系统项目总结

## 项目概述
这是一个基于 Next.js 和 Supabase 构建的中小型企业员工考勤管理系统，实现了企业打卡、请假、加班、旷工的数字化、自动化管理。

## 核心功能

### 三级权限体系
- **普通员工**: 打卡、请假、加班、查看个人记录
- **人事**: 审批申请、员工管理、考勤修正、报表
- **超级管理员**: 全部权限 + 系统设置

### 主要模块
1. **打卡签到** - 上下班在线打卡，自动记录时间
2. **请假管理** - 提交和审批各类请假申请（事假/病假/年假等）
3. **加班管理** - 提交和审批加班申请
4. **考勤记录** - 查看个人每日/月度考勤明细
5. **状态判定** - 自动判断正常/迟到/早退/旷工/加班/休假状态
6. **统计报表** - 可视化考勤数据统计
7. **系统设置** - 班次配置、阈值设置

## 技术架构
- **前端框架**: Next.js 14 (App Router)
- **编程语言**: TypeScript
- **样式**: Tailwind CSS
- **后端服务**: Supabase (PostgreSQL + Auth + RLS)
- **部署平台**: Vercel

## 数据库设计
使用 PostgreSQL 数据库，主要包含以下表：
- `profiles`: 用户资料（关联 auth.users）
- `attendance_records`: 打卡记录
- `leave_requests`: 请假申请
- `overtime_requests`: 加班申请
- `attendance_corrections`: 考勤修正记录
- `shift_config`: 班次配置

所有表都启用了 Row Level Security (RLS)，确保数据安全。

## 关键特性
1. **自动考勤判定**: 系统根据规则自动判断考勤状态
2. **实时审批流程**: 人事可及时处理待办事项
3. **灵活班次配置**: 可自定义上班时间、迟到阈值等
4. **多维度统计**: 提供丰富的考勤数据分析
5. **响应式界面**: 适配不同设备访问

## 部署与启动
项目可通过 Vercel 一键部署，或手动部署到 Vercel 平台。需要配置 Supabase 的 API 凭证环境变量。
